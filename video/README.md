# Manse demo video (Remotion)

Programmatic competition-submission video built with [Remotion](https://www.remotion.dev/).
It follows `docs/DEMO-SCRIPT.md`: nine scenes, ~2:26 total, 1920×1080 @ 30fps,
with TTS voiceover and on-screen captions.

## Commands

```bash
cd video
npm install
npm run dev      # Remotion Studio (interactive preview)
npm run capture  # records the signed-out public Showcase and judge path
npm run render   # renders out/manse-demo.mp4
```

## Structure

- `src/theme.ts` — palette (matches the Showcase's warm cream/coral brand),
  fps, and per-scene durations derived from measured voiceover lengths.
- `src/components.tsx` — shared shell, caption strip, springs, Ken Burns.
- `src/scenes/` — one component per storyboard row of `docs/DEMO-SCRIPT.md`.
- `capture-live.mjs` — Playwright capture with HTTPS/status/page-error checks
  and deterministic completion of the public judge simulator.
- `public/clips/` — signed-out recordings of the deployed Showcase and
  camera-free judge path.
- `public/screens/` — supporting product screenshots.
- `public/audio/scene-N.wav` — voiceover clips.

## Replacing the voiceover

The current narration is macOS TTS (Samantha), generated by `./gen-audio.sh`.
To use a human recording, drop replacement `scene-1.wav … scene-9.wav` files
into `public/audio/`, update `VO_SECONDS` in `src/theme.ts` with the new
durations (`afinfo scene-N.wav`), and re-render. Scene lengths and total
duration adapt automatically.

## Release verification

Run `npm run capture` immediately before the final render. The capture fails if
either public route is unavailable, emits a page error, or the Playground does
not advance through `0/3 → 1/3 → 3/3` and display `Runtime check complete.`
The resulting clips are embedded with Remotion's `<OffthreadVideo>` component.
