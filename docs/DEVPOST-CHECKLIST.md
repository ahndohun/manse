# OpenAI Build Week submission checklist

Verified against the official Devpost overview, rules, resources, and FAQ on 2026-07-21.

## Account and deadline

- [x] Entrant is registered for OpenAI Build Week.
- [ ] Create a Devpost project draft. This changes external state and must be done by or explicitly approved by the entrant.
- [ ] Select one track. Current recommendation: **Developer Tools**, because the core submission is an installable creator/publisher plugin plus an open-source game engine and platform.
- [ ] Submit before **July 21, 2026 at 5:00 PM PT / July 22, 2026 at 9:00 AM KST**.
- [ ] Keep the deployed project available free of charge through the judging period ending August 5, 2026 at 5:00 PM PT.

## Required submission material

- [ ] Working project built with Codex and meaningfully using GPT-5.6
- [ ] English project description and selected category
- [ ] Public YouTube demo, no longer than three minutes
- [ ] Demo audio explains what was built, how Codex was used, and how GPT-5.6 was used
- [ ] Repository URL: public with a relevant license, or privately shared with both required judging addresses
- [ ] README with setup, sample data, testing path, Codex collaboration, key decisions, and GPT-5.6 usage
- [ ] `/feedback` Session ID from the primary task where most core functionality was built

## Additional plugin and developer-tool requirements

- [ ] Installation instructions
- [ ] Supported platforms
- [ ] Test path that does not require judges to rebuild: public Showcase Site, hosted engine/starter playground, and an installable plugin marketplace entry
- [ ] Working demo must match the video and written claims

## Repository and legal checks

- [ ] MIT `LICENSE`
- [ ] Third-party dependency license inventory
- [ ] Asset provenance and license metadata
- [ ] No unlicensed trademarks, copyrighted music, characters, or third-party footage in the video
- [ ] All submission and testing instructions are English
- [ ] Existing and new work are distinguishable through dated commits and the decision log

## Judge experience

- [ ] Public catalog opens without sign-in
- [ ] Engine/starter playground starts without a build step or API key
- [ ] Simulator path works when camera access is unavailable
- [ ] Plugin install can be completed from documented steps
- [ ] Creator can produce, validate, and publish a new game Site, then prepare a Showcase catalog entry
- [ ] README links directly to the live platform, reference game, plugin instructions, demo video, architecture, and privacy model

## Before final submission

- [ ] Run the eligibility and security audit
- [ ] Verify the deployed URL in a clean browser session
- [ ] Verify the reference game after disabling the network following first load
- [ ] Verify the repository is accessible to judges
- [ ] Verify the YouTube video is public and under three minutes
- [ ] Capture the primary task's `/feedback` Session ID
- [ ] Save a local copy of all submitted text and URLs before the deadline
